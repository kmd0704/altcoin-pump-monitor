# Altcoin Pump Monitor v3.6 — 24時間 スマート通知システム

GitHub Actions で **1時間ごと** に CoinGecko を監視。**急騰検知 → 推奨エントリー時刻を計算 → エントリー時刻直前に再通知** する2段階通知。

## ⚡ システムの動き

```
急騰発生(コインが+50%上昇)
  ↓
[Phase 1] 検知 — 1時間以内に GitHub Actions が捕捉
  ↓
急騰開始時刻 T_pump を時間足から逆算
推奨エントリー時刻 T_entry = T_pump + 3時間 を計算
  ↓
🟡 Discord通知:「検知:エントリー予定 19:00 JST」
  ↓
[Phase 2] 待機 — 鎌田さんは何もしない
  ↓
T_entry が近づく(±60分以内)
  ↓
ピーク価格、現在価格、ストップ、TP を再計算
  ↓
🚨 Discord通知:「いますぐエントリー!」+ 執行レシピ
  ↓
鎌田さんが取引所で発注
  ↓
192時間後(8日後)
  ↓
⏰ Discord通知:「決済時刻」
```

## 戦略仕様(v3.6)

```
エントリー条件:
  - 時価総額ランク 250 ≤ rank ≤ 1000
  - 24h急騰 +50% 以上 〜 +200% 未満
  - ステーブル/ラップ/BTC・ETH 除外
  - 直近30日の上昇率 ≤ 100%
  - 直近7日 turnover ≥ 1%
  - 過去48時間内に同銘柄でアラート無し

エントリー後:

  - ストップ: 検知後最高値 × 1.60

  - 保有: 192時間(8日)
```

## 🚀 セットアップ手順(30分で完了)

### Step 1:GitHub アカウント作成

1. https://github.com にアクセス
2. 右上「Sign up」
3. メアド、パスワード、ユーザー名を設定
4. メール認証完了

### Step 2:Discord アカウント + サーバー作成

1. https://discord.com にアクセス → アカウント作成
2. デスクトップアプリをダウンロード(or Webで使う)
3. **スマホアプリも入れる**(通知を受け取る用)
4. 左側「+」ボタン → 「サーバーを作成」 → 「自分用」
   - サーバー名:「アルトpump」など
5. 適当なテキストチャンネル(例:`#alerts`)を作成

### Step 3:Discord Webhook 作成

1. 通知を受け取りたいチャンネルを開く
2. 歯車アイコン(チャンネル設定)→「連携サービス」→「ウェブフック」
3. 「新しいウェブフック」→ 名前を「Pump Monitor」など
4. **「ウェブフックURLをコピー」** ← これを記録
   - 形:`https://discord.com/api/webhooks/123456.../abc...`

### Step 4:GitHub にリポジトリ作成

1. GitHubトップ右上「+」→「New repository」
2. リポジトリ名:`altcoin-pump-monitor`(任意)
3. **Public** を選択(無料Actions無制限。APIキーはSecretsで隠す)
   - Privateにする場合は月2,000分の制約あり(1時間cronなら720分/月でOKだが念のため)
4. 「Create repository」

### Step 5:ファイルをアップロード

#### 方法A:Web上で(初心者向け)
1. リポジトリ画面の「uploading an existing file」リンク
2. このフォルダ(`github_monitor`)の中身を全てドラッグ&ドロップ
   - `monitor.py`
   - `state.json`
   - `README.md`
   - `.gitignore`
3. **`.github/workflows/monitor.yml` も忘れずに**(隠しフォルダ)
   - GitHub Web UI で「Create new file」→ ファイル名に `.github/workflows/monitor.yml` と入力すればフォルダごと作成可能
4. 「Commit changes」

#### 方法B:GitHub Desktop アプリで(推奨)
1. https://desktop.github.com からダウンロード
2. 「Clone」→ 作ったリポジトリを選択
3. このフォルダ(`github_monitor`)の中身を、ローカルにコピー
4. アプリで変更確認 → コミット → プッシュ

### Step 6:GitHub Secrets と Variables を設定

リポジトリページ → **Settings** → 左メニュー **Secrets and variables** → **Actions**

#### 「Secrets」タブ
- 「New repository secret」を**2つ追加**:
  - `CG_API_KEY` = CoinGecko API キー(`CG-xxxxx`)
  - `DISCORD_WEBHOOK` = Step 3 でコピーした URL

#### 「Variables」タブ
- 「New repository variable」を**3つ追加**:
  - `CG_PLAN` = `demo`(または有料契約していれば `pro`)
  - `ACCOUNT_BALANCE` = `100000`(口座残高、円)
  - `POSITION_PCT` = `0.20`(ポジションサイズ、20%)

### Step 7:Actions を有効化&動作確認

1. リポジトリの「Actions」タブを開く
2. 「I understand my workflows, go ahead and enable them」
3. 左側「Altcoin Pump Monitor v3.6」をクリック
4. 右側「Run workflow」→「Run workflow」(初回手動実行)
5. 1-3分待って緑✓が出れば成功

ログを確認すると:
```

[2026-04-25 15:00:01Z] Phase 1: top1000 取得...
[2026-04-25 15:00:15Z]   1000 銘柄取得
[2026-04-25 15:00:15Z]   急騰候補: 0件
[2026-04-25 15:00:15Z] Phase 2: スケジュール処理...
[2026-04-25 15:00:15Z] === 完了:新規検知 0件 ===
```

急騰がない時間帯はこんな感じで0件で終わります(正常)。

### Step 8:本格運用開始

- 以降は **毎時0分に自動実行**(GitHub Actionsが回す)

## 🛠 トラブルシュート

### 「動いているか確認」
- リポジトリの「Actions」タブで実行履歴
- 緑✓=成功、赤✗=失敗、黄=実行中

### 「アラートが来ない」
- Actions のログを確認
- 「急騰候補:0件」=その時間に条件合致が無いだけ(正常、月44件ペース)
- 「ERROR」=設定ミスの可能性

### 「APIエラー」
- CoinGecko ダッシュボードで使用量確認
- 1時間間隔の月次予算想定:
  - 4ページ × 24回/日 × 30日 = **2,880コール**(基本)
  - 候補ごと追加 ≒ 平均 5/日 × 30日 = 150
  - 合計 約 3,030コール/月 → **Demo の 10,000 範囲内** ✓
  - Basicなら余裕

### 「ポジションサイズを変えたい」
- Settings → Variables → `POSITION_PCT` を `0.15` 等に編集

### 「監視を一時停止」
- Actions タブ → 「Altcoin Pump Monitor v3.6」→「・・・」→「Disable workflow」

### 「もっと頻度上げたい(15分間隔)」
- `.github/workflows/monitor.yml` の cron を `*/15 * * * *` に変更
- 月次予算約12,000コール → **Basicプラン契約推奨**

1. **取引判断は最終的に自分で**。アラートはあくまで「条件合致の通知」

4. **タイムゾーン**:通知は JST(日本時間)で表示
